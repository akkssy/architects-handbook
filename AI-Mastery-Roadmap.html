A vector is an ordered list of numbers that represents both magnitude and direction. In ML, vectors represent data points, features, weights, and gradients.

3. The Dot Product: Measuring Similarity

The dot product measures how aligned two vectors are. It's fundamental to attention mechanisms, similarity search, and neural network computations.

Dot Product Formula
a ¬∑ b = Œ£·µ¢ a·µ¢b·µ¢ = |a| |b| cos(Œ∏)
The dot product equals the sum of element-wise products, OR the product of magnitudes times the cosine of the angle between vectors.
Positive Dot Product

Vectors point in similar directions (angle < 90¬∞). High similarity.

Zero Dot Product

Vectors are perpendicular (orthogonal). No correlation.

Negative Dot Product

Vectors point in opposite directions (angle > 90¬∞). Dissimilar.

‚ÑπÔ∏è Cosine Similarity

Cosine similarity = (a ¬∑ b) / (|a| |b|) normalizes the dot product to [-1, 1]. It's used in recommendation systems, document similarity, and word embeddings to compare vectors regardless of their magnitude.

4. Eigenvalues & Eigenvectors: The DNA of Matrices

Eigenvectors are special directions that only get scaled (not rotated) when a matrix transformation is applied. The scaling factor is the eigenvalue.

Eigenvalue Equation

Where A is a matrix, v is an eigenvector, and Œª (lambda) is the corresponding eigenvalue. The matrix A transforms v by simply scaling it by Œª.

5. Singular Value Decomposition (SVD)

SVD decomposes ANY matrix into three special matrices. It's the Swiss Army knife of linear algebra, used in recommendation systems, image compression, and NLP.

SVD Decomposition
A = UŒ£V·µÄ
U = left singular vectors (m√óm orthogonal matrix)
Œ£ = diagonal matrix of singular values (m√ón)
V·µÄ = right singular vectors transposed (n√ón orthogonal matrix)
Python (NumPy)

# Load grayscale image as matrix
img = np.array(Image.open('photo.jpg').convert('L'))

# Compute SVD
U, S, Vt = np.linalg.svd(img, full_matrices=False)

# Compress: keep only top k singular values
k = 50  # Compression rank
compressed = U[:, :k] @ np.diag(S[:k]) @ Vt[:k, :]

# Original: 1000√ó1000 = 1M values
# Compressed: 1000√ó50 + 50 + 50√ó1000 = ~100K values
# 10x compression with minimal quality loss!

  2. Create a small dictionary of word vectors (5-10 words)

import numpy as np

def cosine_similarity(a, b):
    """Compute cosine similarity between two vectors."""
    dot_product = np.dot(a, b)
    norm_a = np.linalg.norm(a)
    norm_b = np.linalg.norm(b)
    return dot_product / (norm_a * norm_b)

# Simple word embeddings (3D for visualization)
words = {
    'king': np.array([0.9, 0.8, 0.1]),
    'queen': np.array([0.85, 0.75, 0.9]),
    'man': np.array([0.7, 0.6, 0.1]),
    'woman': np.array([0.65, 0.55, 0.9]),
    'apple': np.array([0.1, 0.9, 0.5]),
}

def find_most_similar(query, words_dict):
    query_vec = words_dict[query]
    similarities = {}
    for word, vec in words_dict.items():
        if word != query:
            similarities[word] = cosine_similarity(query_vec, vec)
    return max(similarities, key=similarities.get)

print(find_most_similar('king', words))  # 'man'
print(find_most_similar('queen', words)) # 'woman'

1. Probability Fundamentals

Probability quantifies uncertainty. In ML, we use probability to model data distributions, make predictions, and quantify confidence in our results.

Basic Probability Rules
P(A or B) = P(A) + P(B) - P(A and B)
For independent events: P(A and B) = P(A) √ó P(B)
Probability always ranges from 0 (impossible) to 1 (certain)

2. Bayes' Theorem: Updating Beliefs

Bayes' theorem is the foundation of probabilistic ML. It tells us how to update our beliefs when we observe new evidence.

Bayes' Theorem
P(A|B) = P(B|A) √ó P(A) / P(B)
P(A|B) = Posterior (what we want to know)
P(B|A) = Likelihood (how likely is evidence given hypothesis)
P(A) = Prior (initial belief before evidence)
P(B) = Evidence (normalizing constant)
üè• Classic Example: Medical Testing

A disease affects 1% of the population. A test is 99% accurate (both sensitivity and specificity). If you test positive, what's the probability you have the disease?

Intuition says ~99%, but Bayes says ~50%! This is because false positives from the 99% healthy population outnumber true positives from the 1% sick population.

Python 
# Medical test example with Bayes
prior_disease = 0.01        # P(Disease) = 1%
sensitivity = 0.99          # P(Positive|Disease)
specificity = 0.99          # P(Negative|No Disease)

# P(Positive) = P(Pos|Disease)√óP(Disease) + P(Pos|No Disease)√óP(No Disease)
p_positive = sensitivity * prior_disease + (1 - specificity) * (1 - prior_disease)

# Bayes' Theorem
posterior = (sensitivity * prior_disease) / p_positive
print(f"P(Disease|Positive) = {posterior:.1%}")  # ‚âà 50%!
üí° ML Connection

Naive Bayes classifiers use Bayes' theorem for text classification. Bayesian neural networks quantify uncertainty in predictions. Bayesian optimization finds optimal hyperparameters efficiently.

4. Maximum Likelihood Estimation (MLE)

MLE finds the parameters that make observed data most probable. It's how we "fit" models to data.

MLE Principle
Œ∏ÃÇ = argmax L(Œ∏|data) = argmax Œ† P(x·µ¢|Œ∏)
Find parameters Œ∏ that maximize the likelihood of observing our data. In practice, we maximize log-likelihood (sum of log probabilities) for numerical stability.
‚ÑπÔ∏è Why Log-Likelihood?

Multiplying many small probabilities causes numerical underflow. Taking logs converts products to sums: log(a√ób) = log(a) + log(b). This is why neural networks minimize negative log-likelihood (cross-entropy loss).

üèãÔ∏è Exercise Beginner ‚Ä¢ 20 minutes
Simulate the Monty Hall Problem
The famous probability puzzle: You pick a door, the host opens another door revealing a goat, should you switch? Prove the answer through simulation.
  1. Simulate 10,000 games where you always stay with your first choice
  2. Simulate 10,000 games where you always switch
  3. Compare win rates - which strategy wins more?
  4. Explain why switching gives 2/3 probability of winning
Python 
import numpy as np

def monty_hall(switch, n_games=10000):
    wins = 0
    for _ in range(n_games):
        # Car is behind one of 3 doors
        car = np.random.randint(0, 3)
        # Player picks a door
        choice = np.random.randint(0, 3)

        # Host opens a door with a goat (not car, not choice)
        available = [d for d in range(3) if d != car and d != choice]
        host_opens = np.random.choice(available)

        if switch:
            # Switch to the other unopened door
            choice = [d for d in range(3) if d != choice and d != host_opens][0]

        if choice == car:
            wins += 1
    return wins / n_games

print(f"Stay win rate: {monty_hall(switch=False):.1%}")   # ‚âà 33%
print(f"Switch win rate: {monty_hall(switch=True):.1%}")  # ‚âà 67%

Why switching wins 2/3: Your initial choice has 1/3 chance of being correct. The host's reveal doesn't change this‚Äîit just concentrates the remaining 2/3 probability onto the other door!

2. Partial Derivatives & Gradients

When a function has multiple inputs, partial derivatives measure change with respect to one variable while holding others constant. The gradient combines all partial derivatives into a vector.

Gradient Vector
‚àáf = [‚àÇf/‚àÇx‚ÇÅ, ‚àÇf/‚àÇx‚ÇÇ, ..., ‚àÇf/‚àÇx‚Çô]
The gradient points in the direction of steepest increase. Its magnitude tells us how steep that increase is.
üß≠ Gradient Intuition

Imagine standing on a hilly landscape. The gradient tells you which direction is steepest uphill. To minimize loss (go downhill), we move in the opposite direction of the gradient‚Äîthis is gradient descent!

Python (NumPy) 
# Gradient of f(x,y) = x¬≤ + y¬≤
def f(x, y):
    return x**2 + y**2

def gradient_f(x, y):
    df_dx = 2 * x  # ‚àÇf/‚àÇx
    df_dy = 2 * y  # ‚àÇf/‚àÇy
    return np.array([df_dx, df_dy])

# At point (3, 4)
grad = gradient_f(3, 4)  # [6, 8]
print(f"Gradient: {grad}")
print(f"Magnitude: {np.linalg.norm(grad)}")  # 10
3. The Chain Rule: Backbone of Backpropagation

The chain rule lets us differentiate composite functions. It's the mathematical foundation of backpropagation‚Äîhow neural networks learn.

Chain Rule
d/dx[f(g(x))] = f'(g(x)) ¬∑ g'(x)
To differentiate a composition, multiply the derivative of the outer function (evaluated at the inner) by the derivative of the inner function.
Chain Rule in Neural Networks
Input ‚Üí [Layer 1] ‚Üí [Layer 2] ‚Üí [Layer 3] ‚Üí Loss
  x        h‚ÇÅ          h‚ÇÇ          y         L

Forward pass: compute outputs left to right
Backward pass: compute gradients right to left

‚àÇL/‚àÇw‚ÇÅ = ‚àÇL/‚àÇy ¬∑ ‚àÇy/‚àÇh‚ÇÇ ¬∑ ‚àÇh‚ÇÇ/‚àÇh‚ÇÅ ¬∑ ‚àÇh‚ÇÅ/‚àÇw‚ÇÅ
         ‚Üë        ‚Üë         ‚Üë         ‚Üë
      Chain of derivatives multiplied together

# PyTorch autograd handles chain rule automatically!

# Composite function: f(g(x)) where g(x)=x¬≤, f(u)=sin(u)

# Compute gradient via backpropagation

# Chain rule: d/dx[sin(x¬≤)] = cos(x¬≤) ¬∑ 2x

Modern deep learning frameworks (PyTorch, TensorFlow) implement automatic differentiation. They build a computational graph and apply the chain rule automatically. Understanding the chain rule helps you debug gradient issues and design custom layers.

4. Gradient Descent: How Models Learn

Gradient descent iteratively updates parameters to minimize a loss function. It's the optimization algorithm that powers virtually all neural network training.

Gradient Descent Update
Œ∏ = Œ∏ - Œ± ¬∑ ‚àáL(Œ∏)
Œ∏ = parameters (weights)
Œ± = learning rate (step size)
‚àáL(Œ∏) = gradient of loss with respect to parameters
Python 
# Simple gradient descent to find minimum of f(x) = x¬≤
def gradient_descent(start, learning_rate, n_iterations):
    x = start
    history = [x]

    for _ in range(n_iterations):
        gradient = 2 * x  # derivative of x¬≤
        x = x - learning_rate * gradient
        history.append(x)

    return x, history

# Start at x=5, find minimum
minimum, path = gradient_descent(start=5, learning_rate=0.1, n_iterations=20)
print(f"Minimum found at x = {minimum:.6f}")  # ‚âà 0
‚ö†Ô∏è Learning Rate Matters

Too high: Overshoots minimum, may diverge. Too low: Converges very slowly. Finding the right learning rate is crucial‚Äîthis is why techniques like learning rate scheduling and adaptive optimizers (Adam, RMSprop) exist.

4. The Bias-Variance Tradeoff

Understanding overfitting and underfitting is crucial for building models that generalize well to new data.

Underfitting (High Bias)

Model too simple. Poor performance on both training and test data. Solution: more features, complex model.

Good Fit

Model captures patterns without memorizing noise. Good performance on both training and test data.

Overfitting (High Variance)

Model memorizes training data. Great on training, poor on test. Solution: regularization, more data.

import numpy as np

# XOR data
X = np.array([[0,0], [0,1], [1,0], [1,1]])
y = np.array([[0], [1], [1], [0]])

# Initialize weights
np.random.seed(42)
W1 = np.random.randn(2, 4) * 0.5
b1 = np.zeros((1, 4))
W2 = np.random.randn(4, 1) * 0.5
b2 = np.zeros((1, 1))

def relu(x): return np.maximum(0, x)
def relu_deriv(x): return (x > 0).astype(float)
def sigmoid(x): return 1 / (1 + np.exp(-x))

# Training
lr = 0.5
for epoch in range(10000):
    # Forward
    z1 = X @ W1 + b1
    a1 = relu(z1)
    z2 = a1 @ W2 + b2
    a2 = sigmoid(z2)

    # Backward
    dz2 = a2 - y
    dW2 = a1.T @ dz2 / 4
    db2 = dz2.mean(axis=0)
    dz1 = (dz2 @ W2.T) * relu_deriv(z1)
    dW1 = X.T @ dz1 / 4
    db1 = dz1.mean(axis=0)

    # Update
    W2 -= lr * dW2; b2 -= lr * db2
    W1 -= lr * dW1; b1 -= lr * db1

print("Predictions:", (a2 > 0.5).astype(int).flatten())

import torch.nn as nn

class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(3, 32, 3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),
            nn.Conv2d(32, 64, 3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),
        )
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(64 * 56 * 56, 512),
            nn.ReLU(),
            nn.Linear(512, num_classes)
        )

    def forward(self, x):
        x = self.features(x)
        return self.classifier(x)
4. Transfer Learning: Don't Train from Scratch

Pre-trained models (trained on ImageNet) already know how to detect edges, textures, and objects. Fine-tune them for your specific task.

Python (PyTorch) 
import torchvision.models as models

# Load pre-trained ResNet
model = models.resnet50(pretrained=True)

# Freeze all layers
for param in model.parameters():
    param.requires_grad = False

# Replace final layer for your task
num_classes = 5  # Your number of classes
model.fc = nn.Linear(model.fc.in_features, num_classes)

# Only train the new layer
optimizer = torch.optim.Adam(model.fc.parameters(), lr=0.001)

3. Attention: The Key Innovation

Attention allows models to focus on relevant parts of the input when producing each output. It's the core mechanism behind Transformers.

Scaled Dot-Product Attention
Attention(Q, K, V) = softmax(QKT / ‚àödk)V
Q = Query (what we're looking for)
K = Key (what we're matching against)
V = Value (what we retrieve)
dk = dimension of keys (for scaling)
Self-Attention Example
Input: "The cat sat on the mat"

When processing "sat", attention weights might be:
  The   cat   sat   on   the   mat
  0.05  0.35  0.30  0.10  0.05  0.15
        ‚Üë
   "cat" gets high attention because
   it's the subject of "sat"
Python (PyTorch) 
import torch
import torch.nn.functional as F

def attention(Q, K, V):
    """Scaled dot-product attention"""
    d_k = Q.size(-1)
    scores = torch.matmul(Q, K.transpose(-2, -1)) / (d_k ** 0.5)
    weights = F.softmax(scores, dim=-1)
    return torch.matmul(weights, V), weights

# Example

Q(s,a) = Action-value function (expected return from state s, action a)
Œ≥ = Discount factor (0-1, how much to value future rewards)
2. Q-Learning: Learning from Experience

Q-Learning learns the optimal action-value function by iteratively updating Q-values based on observed rewards and estimated future values.

Q-Learning Update Rule
Q(s,a) ‚Üê Q(s,a) + Œ±[r + Œ≥ max_a' Q(s',a') - Q(s,a)]
Œ± = Learning rate
r = Observed reward
s' = Next state
max_a' Q(s',a') = Best estimated future value
Python 
import numpy as np
import gymnasium as gym

# Simple Q-Learning
env = gym.make('FrozenLake-v1')
Q = np.zeros([env.observation_space.n, env.action_space.n])
alpha, gamma, epsilon = 0.1, 0.99, 0.1

for episode in range(10000):
    state, _ = env.reset()
    done = False

    while not done:
        # Epsilon-greedy action selection
        if np.random.random() < epsilon:
            action = env.action_space.sample()
        else:
            action = np.argmax(Q[state])

        next_state, reward, done, _, _ = env.step(action)

        # Q-Learning update
        Q[state, action] += alpha * (
            reward + gamma * np.max(Q[next_state]) - Q[state, action]
        )
        state = next_state
3. Policy Gradients: Learning Policies Directly

Instead of learning value functions, policy gradient methods directly optimize the policy parameters to maximize expected return.

Value-Based (DQN)

Learn Q(s,a), derive policy from it. Works well for discrete actions.

Policy-Based (REINFORCE)

Learn œÄ(a|s) directly. Works for continuous actions, can learn stochastic policies.

Actor-Critic (PPO, A2C)

Best of both: policy (actor) + value function (critic). Most popular in practice.

üí° PPO: The Go-To Algorithm

Proximal Policy Optimization (PPO) is the most widely used RL algorithm. It's stable, sample-efficient, and works across many domains. It's also the algorithm used for RLHF in ChatGPT!

2. RLHF: Aligning LLMs with Human Preferences

Reinforcement Learning from Human Feedback (RLHF) trains models to produce outputs that humans prefer.

Step 1: Collect Preferences

Humans compare model outputs and indicate which is better. This creates a preference dataset.

Step 2: Train Reward Model

A reward model learns to predict human preferences, scoring outputs by quality.

Step 3: Optimize with PPO

The LLM is fine-tuned using PPO to maximize the reward model's scores.

üí° DPO: A Simpler Alternative

Direct Preference Optimization (DPO) skips the reward model entirely, directly optimizing the LLM on preference data. It's simpler, more stable, and increasingly popular.

4. LoRA: Efficient Fine-tuning

Low-Rank Adaptation (LoRA) enables fine-tuning large models with minimal compute by only training small adapter matrices.

LoRA Concept
W' = W + BA
W = Original frozen weights (billions of parameters)
B, A = Low-rank adapter matrices (millions of parameters)
W' = Effective weights after adaptation

Instead of updating all of W, we only train B and A!
‚ö†Ô∏è QLoRA for Consumer Hardware

QLoRA combines LoRA with 4-bit quantization, enabling fine-tuning of 65B+ parameter models on a single GPU. Essential for running LLMs locally!

2. Experiment Tracking with MLflow

Track experiments, compare runs, and manage models with MLflow - the most popular open-source MLOps platform.

Python 
import mlflow
import mlflow.sklearn
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import accuracy_score

# Set experiment
mlflow.set_experiment("fraud-detection")

# Start a run
with mlflow.start_run():
    # Log parameters
    mlflow.log_param("n_estimators", 100)
    mlflow.log_param("max_depth", 10)

    # Train model
    model = RandomForestClassifier(n_estimators=100, max_depth=10)
    model.fit(X_train, y_train)

    # Log metrics
    predictions = model.predict(X_test)
    accuracy = accuracy_score(y_test, predictions)
    mlflow.log_metric("accuracy", accuracy)

    # Log model
    mlflow.sklearn.log_model(model, "model")

    # Register model
    mlflow.register_model(
        f"runs:/{mlflow.active_run().info.run_id}/model",
        "fraud-detector"
    )

Different deployment patterns for different use cases.

REST API (FastAPI)

Simple, synchronous. Good for low-latency, single predictions. Easy to deploy and scale.

Batch Inference

Process large datasets offline. Cost-effective for non-real-time use cases.

Streaming (Kafka)

Real-time predictions on event streams. Complex but powerful for high-throughput.

# Simple Model Serving with FastAPI
from fastapi import FastAPI
import mlflow
from pydantic import BaseModel

app = FastAPI()

# Load model at startup
model = mlflow.sklearn.load_model("models:/fraud-detector/Production")

class PredictionRequest(BaseModel):
    features: list[float]

@app.post("/predict")
def predict(request: PredictionRequest):
    prediction = model.predict([request.features])
    return {"prediction": int(prediction[0])}
4. Model Monitoring & Data Drift

Models degrade over time as data distributions change. Monitoring is essential for production ML.

‚ö†Ô∏è Types of Drift

Data Drift: Input distribution changes (e.g., new user demographics)
Concept Drift: Relationship between inputs and outputs changes (e.g., fraud patterns evolve)
Model Drift: Model performance degrades over time
