üß© Code Chunker

Intelligently splits code into semantic chunks for better search results.

Python Files (AST-based)

  • Extracts individual functions
  • Extracts classes with methods
  • Preserves module-level code
  • Maintains line numbers

Other Languages (Line-based)

  • Configurable chunk size (default: 50 lines)
  • Overlapping chunks for context preservation
  • Language detection from file extension

üìä Codebase Indexer

Creates and manages the vector database index.

Features

  • Local embeddings - No API calls needed
  • Persistent storage - Index saved to disk
  • Incremental updates - Re-index changed files
  • Smart filtering - Ignores node_modules, .git, etc.

Supported Extensions

.py .js .ts .jsx .tsx .java .go .rs .rb .php .c .cpp .h .cs .swift .kt .scala .sql .yaml .json .md .html .css .vue .svelte

üîé Semantic Search

Find code using natural language descriptions.

How It Works

  1. Query text is converted to an embedding vector
  2. ChromaDB finds similar vectors in the index
  3. Results are ranked by similarity score
  4. Filters are applied (file path, language)

Output Formats

  • console - Rich formatted terminal output
  • json - Structured data for automation
  • context - Formatted for LLM prompts

‚öôÔ∏è Configuration

# config.yaml - Retrieval Settings
retrieval:
  # Embedding model (runs locally, ~90MB download)
  embedding_model: "all-MiniLM-L6-v2"

  # Directory to store the index
  persist_directory: ".ai-assistant-index"

  # ChromaDB collection name
  collection_name: "codebase"

  # Maximum lines per chunk (for non-AST chunking)
  chunk_size: 50

  # Overlap between chunks
  chunk_overlap: 10

  # Skip files larger than this (KB)
  max_file_size_kb: 1024
